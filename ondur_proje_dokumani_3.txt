## 1. Proje Tanımı

Ondur, ziraat mühendisleri, çiftçiler, ilaç/gübre üreticileri ve bayiileri bir araya getiren tarımsal danışmanlık ve reçete yönetim sistemidir.

**Teknoloji:**
- Backend: Django 6 + Django REST Framework (DRF)
- Frontend: React (henüz başlanmadı)
- Veritabanı: SQLite (geliştirme) → PostgreSQL (yayın)
- Mobil: İleride React Native

**Kurulum:**
```
cd C:\ondur
venv\Scripts\activate
python manage.py runserver
```

**Admin paneli:** http://127.0.0.1:8000/admin

## 2. Kullanıcı Rolleri

```python
class Rol(models.TextChoices):
    YONETICI  = 'yonetici'
    YARDIMCI  = 'yardimci'
    MUHENDIS  = 'muhendis'
    CIFTCI    = 'ciftci'
    URETICI   = 'uretici'
    BAYII     = 'bayii'
```

- Tüm roller kendisi kaydolur → direkt aktif
- Çiftçi kaydolunca 8 haneli kimlik kodu otomatik üretilir (Luhn)
- Boşluklu gösterim: 873 645 13

---

## 3. İlişkiler

**Mühendis — Çiftçi:**
- Mühendis kimlik koduyla çiftçiyi arar
- İşletme seçer, talep açar (MuhendisIsletme → bekliyor)
- Çiftçi her işletme için ayrı onaylar/reddeder

**Çiftçi — Bayii:**
- İkisi de başlatabilir
- Kim başlatmadıysa o onaylar
- Onaylanan ilişkide bayii reçete analizini görür

**Mühendis — Bayii:**
- Bağımsız mühendis → ilişki yok
- Bayii mühendisi → MuhendisBayii kaydı var

## 4. Tüm Modeller

### accounts/
- Kullanici (AbstractUser) — rol, telefon(unique), il, ilce
- YardimciYetki — yardimci(FK), yetki, aktif, veren(FK)
- MuhendisIlce — muhendis(FK), il, ilce, aktif

### ciftci/
- Urun — ad(unique), aktif
- UrunCesit — urun(FK), ad, aktif
- Ciftci — kullanici(OneToOne), kimlik_kodu(8 hane), ad, soyad, cks_no, mahalle, ilce, il, telefon, email, aktif
- Isletme — ciftci(FK), ad, tur, sera_tip, urun(FK), cesit(FK), alan_dekar, ekim_tarihi, enlem, boylam, aktif
- MuhendisIsletme — muhendis(FK), isletme(FK), durum, talep_tarihi, yanit_tarihi
- **CifciBayii — HENÜZ EKLENMEDİ** (katalog.Bayii hazır, eklenebilir)

### katalog/
- EtkenMadde — ad(unique), cas_no, aktif, onaylandi
- Uretici — kullanici(OneToOne), firma_adi, vergi_no(unique), adres, yetkili
- Bayii — kullanici(OneToOne), firma_adi, ruhsat_no, il, ilce, telefon
- Ilac — uretici(FK), ticari_ad, kategori, formulasyon, ruhsat_no, phi_gun, bekleme_suresi, endikasyon, doz_min, doz_max, doz_birimi, uygulama_yontemi, kullanim_tavsiyesi, notlar, ambalaj_hacmi, ambalaj_birimi, ambalaj_birim, aktif, onaylandi
- IlacEtkenMadde — ilac(FK), etken_madde(FK-PROTECT), oran, miktar, miktar_birim
- Gubre — uretici(FK), ticari_ad, tur, formulasyon, doz_min, doz_max, doz_birimi, uygulama_yontemi, bekleme_suresi, kullanim_tavsiyesi, notlar, ambalaj_hacmi, ambalaj_birimi, ambalaj_birim, aktif, onaylandi
- GubreEtkenMadde — gubre(FK), etken_madde(FK-PROTECT), oran, miktar, miktar_birim
- BayiiUrun — bayii(FK), ilac(FK-null), gubre(FK-null), aktif
- MuhendisBayii — muhendis(FK), bayii(FK), baslangic, aktif

### recete/
- Recete — isletme(FK), muhendis(FK), tani, tarih, uygulama_yontemi, durum, muhendis_notu, hatirlatma, ciftciye_not, duzenleme_sayisi
- UygulamaAdimi — recete(FK), sira_no, tip, tanim, uygulama_tarihi, tamamlandi, notlar
- UygulamaAdimKalemi — adim(FK), ilac(FK-null), gubre(FK-null), doz_dekar, birim
- ReceteVersiyon — recete(FK), versiyon_no, tani, uygulama_yontemi, notlar, kalemler(JSON), duzenleme_tarihi, duzenleyen(FK)
- ReceteYorum — recete(FK), yazan(FK), tip, ust_yorum(FK-self), metin
- ReceteFotograf — recete(FK), yukleyen(FK), fotograf, tip, aciklama

### ziyaret/
- Ziyaret — isletme(FK), muhendis(FK), tur, tarih, saat, sure_dk, adres, notlar, tamamlandi

## 5. Reçete Yazma Akışı

```
Adım 1 — Tanı: hastalık seç, tanı notu, fotoğraf
Adım 2 — Uygulama Adımları:
  Her adımda: tip, tarih, ürünler (doz × alan = toplam)
  PHI uyarısı otomatik
Sol Panel (sabit):
  Mühendis Notu → sadece mühendis
  Hatırlatma    → bildirim
  Çiftçiye Not  → çiftçi görür
Adım 3 — Özet & Onay: 3 onay kutusu, taslak/onaylı
```
