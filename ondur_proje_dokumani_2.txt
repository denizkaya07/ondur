## 1. Proje Tanımı

Ondur, ziraat mühendisleri, çiftçiler, ilaç/gübre üreticileri ve bayiileri bir araya getiren tarımsal danışmanlık ve reçete yönetim sistemidir.

**Teknoloji:**
- Backend: Django 6 + Django REST Framework (DRF)
- Frontend: React (henüz başlanmadı)
- Veritabanı: SQLite (geliştirme) → PostgreSQL (yayın)
- Mobil: İleride React Native

**Dizin yapısı:**

## 3. Kullanıcı Rolleri

```python
class Rol(models.TextChoices):
    YONETICI  = 'yonetici',  'Yönetici'
    YARDIMCI  = 'yardimci',  'Yardımcı'
    MUHENDIS  = 'muhendis',  'Mühendis'
    CIFTCI    = 'ciftci',    'Çiftçi'
    URETICI   = 'uretici',   'Üretici'
    BAYII     = 'bayii',     'Bayii'
```

### Yönetici
- Sistemi yöneten kişi (sistem sahibi)
- Tam yetki, Django Admin üzerinden çalışır
- Yardımcı ekler, yetkilerini belirler

### Yardımcı
- Yöneticinin verdiği yetkilerle sınırlı
- Yetkiler: muhendis_ekle, muhendis_duzenle, muhendis_pasif, katalog_ekle, katalog_duzenle, katalog_sil, raporlar, kullanici_yon

### Mühendis
- Kendisi kaydolur → direkt aktif
- Kayıt olurken: ad, soyad, telefon, şifre, çalıştığı ilçeler (birden fazla)
- İki tip: bağımsız veya bayii bünyesinde
- Reçete yazar, ziyaret planlar, danışan yönetir

### Çiftçi
- Kendisi kaydolur → direkt aktif
- Sistem 8 haneli kimlik kodu üretir (Luhn algoritması)
- Boşluklu gösterim: 873 645 13
- İşletmelerini kendisi ekler

### Üretici
- Kendisi kaydolur → direkt aktif
- İlaç ve gübre kataloğunu girer

### Bayii
- Kendisi kaydolur → direkt aktif
- Hangi ürünleri sattığını girer
- Bağlı çiftçilerin reçete analizini görür

---

## 4. İlişkiler

### Mühendis — Çiftçi (İşletme Bazında)
```
Mühendis kimlik koduyla çiftçiyi arar
→ İşletmelerini görür, seçer, talep açar
→ MuhendisIsletme kaydı oluşur (durum=bekliyor)
→ Çiftçi uygulamasından her işletme için ayrı onaylar/reddeder
→ Onaylanan işletmeler için mühendis reçete yazabilir
```

### Çiftçi — Bayii (İki Yönlü)
```
Bayii başlatırsa → çiftçi onaylar
Çiftçi başlatırsa → bayii onaylar
Onaylanan ilişkide bayii çiftçinin reçetelerindeki
ürünleri toplam miktar bazında görebilir
```

### Mühendis — Bayii
```
Bağımsız mühendis → bayii ilişkisi yok
Bayii mühendisi   → MuhendisBayii kaydı var
```

## 5. Modeller

### accounts/models.py
```python
Kullanici(AbstractUser)
    rol, telefon(unique), il, ilce

YardimciYetki
    yardimci(FK), yetki, aktif, veren(FK), tarih
    unique: (yardimci, yetki)

MuhendisIlce
    muhendis(FK), il, ilce, aktif
    unique: (muhendis, il, ilce)
```

### ciftci/models.py
```python
Urun
    ad(unique), aktif

UrunCesit
    urun(FK), ad, aktif
    unique: (urun, ad)

Ciftci
    kullanici(OneToOne), kimlik_kodu(8 hane, unique, otomatik)
    ad, soyad, cks_no(opsiyonel), mahalle, ilce, il
    telefon, email, aktif, kayit

Isletme
    ciftci(FK), olusturan(FK)
    ad, tur(sera/acik_tarla/meyve_bahce/zeytinlik/diger)
    sera_tip(naylon/cam/policarbon/net/diger) — sadece tur=sera ise
    urun(FK), cesit(FK)
    alan_dekar, ekim_tarihi
    enlem, boylam (GPS)
    aktif, olusturma

MuhendisIsletme
    muhendis(FK), isletme(FK)
    durum(bekliyor/onaylandi/reddedildi/iptal)
    talep_tarihi, yanit_tarihi
    unique: (muhendis, isletme)

# CifciBayii → katalog.Bayii hazır olunca eklenecek
```

### katalog/models.py
```python
EtkenMadde
    ad(unique), cas_no, aktif, onaylandi

Uretici
    kullanici(OneToOne), firma_adi, vergi_no(unique), adres, yetkili

Bayii
    kullanici(OneToOne), firma_adi, ruhsat_no, il, ilce, telefon

Ilac
    uretici(FK), ticari_ad, kategori, formulasyon, ruhsat_no
    phi_gun, bekleme_suresi, endikasyon
    doz_min, doz_max, doz_birimi, uygulama_yontemi
    kullanim_tavsiyesi, notlar
    ambalaj_hacmi, ambalaj_birimi, ambalaj_birim
    aktif, onaylandi, giris, guncelleme

IlacEtkenMadde
    ilac(FK), etken_madde(FK-PROTECT)
    oran(%), miktar, miktar_birim
    unique: (ilac, etken_madde)

Gubre
    uretici(FK), ticari_ad, tur, formulasyon
    doz_min, doz_max, doz_birimi, uygulama_yontemi
    bekleme_suresi, kullanim_tavsiyesi, notlar
    ambalaj_hacmi, ambalaj_birimi, ambalaj_birim
    aktif, onaylandi, giris, guncelleme

GubreEtkenMadde
    gubre(FK), etken_madde(FK-PROTECT)
    oran(%), miktar, miktar_birim
    unique: (gubre, etken_madde)

BayiiUrun
    bayii(FK), ilac(FK-null), gubre(FK-null), aktif
    # ilac veya gubre'den biri dolu olmalı

MuhendisBayii
    muhendis(FK), bayii(FK), baslangic, aktif
    unique: (muhendis, bayii)
```

### recete/models.py
```python
Recete
    isletme(FK), muhendis(FK)
    tani, tarih, uygulama_yontemi
    durum(taslak/onaylandi/iptal)
    muhendis_notu    ← sadece mühendis görür
    hatirlatma       ← bildirim için
    ciftciye_not     ← çiftçi görür
    duzenleme_sayisi, olusturma, guncelleme

UygulamaAdimi
    recete(FK), sira_no, tip(sulama/ilaclama/gubreleme/diger)
    tanim, uygulama_tarihi, tamamlandi, notlar
    unique: (recete, sira_no)
    ordering: sira_no

UygulamaAdimKalemi
    adim(FK), ilac(FK-null), gubre(FK-null)
    doz_dekar, birim
    property: toplam_miktar = doz_dekar × isletme.alan_dekar

ReceteVersiyon
    recete(FK), versiyon_no, tani, uygulama_yontemi
    notlar, kalemler(JSONField)
    duzenleme_tarihi, duzenleyen(FK), duzenleme_notu
    unique: (recete, versiyon_no)

ReceteYorum
    recete(FK), yazan(FK)
    tip(soru/yorum/yanit), ust_yorum(FK-self-null)
    metin, olusturma, guncelleme

ReceteFotograf
    recete(FK), yukleyen(FK)
    fotograf(ImageField), tip(hastalik/uygulama/genel)
    aciklama, olusturma
```

### ziyaret/models.py
```python
Ziyaret
    isletme(FK), muhendis(FK)
    tur(saha/danismanlik/hasat/recete/numune/planlama)
    tarih, saat, sure_dk, adres, notlar
```

## 6. Reçete Yazma Akışı

```
Adım 1 — Tanı
├── Danışan seçilir (kimlik koduyla veya listeden)
├── İşletme seçilir
├── Hastalık/zararlı seçilir
├── Tanı notu yazılır
└── Fotoğraf eklenebilir

Adım 2 — Uygulama Adımları
├── Her sulama/ilaçlama ayrı kart
├── Her adımda ürün seçilir, dekara doz girilir
├── Sistem otomatik hesaplar (doz × alan = toplam)
└── PHI uyarısı otomatik gösterilir

Sol Panel (sabit)
├── Mühendis Notu    → sadece mühendis görür
├── Hatırlatma       → sistem bildirim gönderir
└── Çiftçiye Not     → çiftçi görür, WhatsApp'ta çıkar

Adım 3 — Onay & Özet
├── Tüm adımlar listelenir
├── 3 onay kutusu
└── Taslak veya Onaylı kaydedilir
```
